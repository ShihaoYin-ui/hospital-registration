# 云医院挂号系统（后端）使用教程

## 一、项目概述

这是一个基于 **Spring Boot 3.5.14 + MyBatis-Plus + MySQL** 的后端项目，提供医院在线挂号的 REST API 接口。

## 二、技术栈

| 组件 | 版本/说明 |
|------|----------|
| JDK | **17+** |
| Spring Boot | 3.5.14 |
| ORM | MyBatis-Plus 3.5.12 |
| 数据库 | MySQL 8.0 |
| 构建工具 | Maven |
| 端口 | **8080** |

## 三、启动步骤

### 1. 创建数据库

先在 MySQL 中创建数据库并执行建表语句：

```sql
CREATE DATABASE hospital_db DEFAULT CHARACTER SET utf8mb4;
USE hospital_db;
-- 然后复制 table.sql 中所有 CREATE TABLE 语句执行
```

### 2. 修改数据库连接配置

如果本地 MySQL 的 root 密码不是 `123456`，修改 `back-end/src/main/resources/application.properties`：

```properties
spring.datasource.username=root
spring.datasource.password=你的密码
```

### 3. 启动项目

在 `back-end` 目录下执行：

```bash
# 方式一：直接运行
mvn spring-boot:run

# 方式二：先打包再运行
mvn clean package -DskipTests
java -jar target/back-end-0.0.1-SNAPSHOT.jar
```

启动成功后访问 `http://localhost:8080`。

## 四、API 接口一览

### 用户相关 `/api/user`

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/user/login` | 手机号+密码登录 |
| POST | `/api/user/register` | 注册 |
| PUT | `/api/user/password` | 修改密码 |
| GET | `/api/user/info` | 查询用户信息 |

### 就诊人管理 `/api/patient`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/patient/list` | 用户的就诊人列表 |
| GET | `/api/patient/default` | 获取默认就诊人 |
| POST | `/api/patient/add` | 添加就诊人 |
| PUT | `/api/patient/update` | 编辑就诊人 |
| DELETE | `/api/patient/delete` | 删除就诊人 |
| PUT | `/api/patient/setDefault` | 设为默认就诊人 |

### 科室类型 `/api/type`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/type/root` | 获取顶级科室分类 |
| GET | `/api/type/children` | 获取子分类 |

### 科室信息 `/api/department`

### 医生信息 `/api/doctor`

| 方法 | 路径 | 说明 |
|------|------|------|

### 排班查询 `/api/schedule`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/schedule/available` | 某天某医生的可用号源 |
| GET | `/api/schedule/range` | 某段时间的号源 |

### 预约挂号 `/api/appointment`

| 方法 | 路径 | 说明 |
|------|------|------|

| GET | `/api/appointment/patient` | 就诊人的预约列表 |

### 就诊记录 `/api/visit`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/visit/byAppointment` | 按预约查就诊记录 |
| GET | `/api/visit/byPatient` | 按就诊人查就诊记录 |

## 五、接口返回格式

所有接口统一返回 JSON：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

- `code` 为 200 表示成功，500 表示失败
- `data` 为实际返回数据，失败时为 `null`

## 六、注意事项

1. **数据库必须先建好**，否则启动会报数据库连接错误
2. 项目**没有提供初始化数据**（INSERT 语句），只有建表 DDL，需要手动插入科室类型、科室、医生、固定排班等基础数据才能正常使用
3. 预约时有**并发控制**（对 `dynamic_schedule` 表行锁定），防止超售号源
4. 没有引入认证框架（如 Spring Security/JWT），登录只是简单的手机号+密码比对，返回用户 ID
