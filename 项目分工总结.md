# 项目分工总结 — 尹仕豪

## 一、API 调试与后端修复

### 1.1 实体类驼峰命名统一重构

发现并修复 MyBatis Plus `map-underscore-to-camel-case=true` 全局配置与 snake_case 实体字段的冲突问题：数据库列名被转为驼峰后找不到对应 setter，导致接口返回全部 null。

- **修复实体（10个）**：User、Patient、Doctor、Department、Type、Appointment、DynamicSchedule、FixedSchedule、VisitRecord、FeeConfig
- **修复方式**：所有字段改为驼峰命名，添加 `@TableField` 显式映射数据库列名
- **同步修复 Service 层**：AppointmentServiceImpl、PatientServiceImpl、UserServiceImpl 中所有 snake_case 方法调用

### 1.2 注册接口修复

- 前端注册时空邮箱导致重复冲突 → 后端加 `!email.isEmpty()` 判断
- 注册后自动添加为默认就诊人（前端链式调用 addPatient）

### 1.3 DepartmentMapper SQL 修复

- 列名 `type` → `type_id`，解决 500 内部错误

### 1.4 排班数据生成与导入

- 生成 120 条排班数据（20 医生 × 3 天 × 2 时段）
- 解决外键约束问题（先删 visit_record → appointment → dynamic_schedule）
- 验证 API 返回正确

### 1.5 数据整合

- 将 `测试数据.sql`、`数据补充.sql`、`排班补充.sql` 合并为统一 `data.sql`
- 清理多余空行
