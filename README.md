# 云医院挂号系统

## 一、项目概述
云医院挂号系统是一个基于B/S架构的在线医疗预约平台，旨在解决传统医院挂号排队时间长、号源信息不透明等问题。系统支持患者在线浏览科室、医生信息，根据医生排班快速预约挂号，并管理个人就诊记录。系统分为患者端（Web/App）与后台管理端，数据库采用MySQL，后端可选用Spring Boot或端云一体化开发。

## 二、核心功能模块
1. 用户管理：注册、登录、个人信息维护。
2. 就诊人管理：支持添加/编辑/删除就诊人（本人及家属），设置默认就诊人。
3. 科室与医生浏览：按科室类型（内、外、妇、儿等）导航，查看科室详情、医生简介、专业擅长、职称等。
4. 排班与号源：基于固定排班模板（每周规律）动态生成每日号源；展示医生出诊时间、剩余号数、挂号费。
5. 预约挂号：选择就诊人、医生、时段后锁定号源，生成预约订单（支持在线支付或到院支付）。
6. 订单管理：查询预约记录，支持就诊前规定时间内取消预约并释放号源。
7. 就诊签到：患者到院后凭预约码签到，更新签到时间。
8. 就诊记录：医生接诊后填写诊断、处方，生成就诊记录，关联预约单。
9. 后台管理：配置科室/医生/排班模板/费用规则，查看预约统计报表。

## 三、数据库核心表设计（基于已提供SQL）
- `user`：用户账号信息（身份证、手机、邮箱、密码、状态）
- `patient`：就诊人（与user关联，支持多就诊人）
- `type`：科室分类（大科类型）
- `department`：科室（所属类型、负责人、位置、介绍）
- `doctor`：医生（所属科室、职称、专业、从业年限、履历）
- `fixed_schedule`：固定排班模板（医生每周出诊规律、时段、总号源数、挂号费）
- `dynamic_schedule`：动态排班（根据模板生成具体某一天的号源，含剩余号源、停诊状态）
- `appointment`：预约记录（关联用户、就诊人、科室、医生、动态排班，记录状态、支付费用、签到/取消时间）
- `visit_record`：就诊记录（关联预约，记录诊断、处方、备注）
- `fee_config`：费用配置（按科室、医生职称、挂号类型、有效期设置挂号费）

## 四、核心业务流程
1. 用户登录 → 添加就诊人 → 选择科室/医生 → 查看排班日历及剩余号源。
2. 确认预约 → 系统检查号源并发锁 → 扣减动态排班剩余号源 → 生成预约记录（状态：待支付/已预约）。
3. 支付（可选） → 预约生效 → 就诊当日用户签到 → 医生接诊并填写就诊记录 → 状态流转为“已完成”。
4. 若用户提前取消预约 → 释放号源 → 预约状态变更为“已取消”。
5. 若用户未在约定时间内签到 → 预约状态变更为“爽约”，号源不释放。

## 五、技术实现
- 前端：HarmonyOS ArkTS + ArkUI（或Vue/React Web端）
- 后端：Spring Boot + MyBatis + MySQL，或使用端云一体化（云函数+云数据库）
- 并发控制：预约时对`dynamic_schedule`表行锁定（SELECT FOR UPDATE）或乐观锁（version字段）
- 定时任务：每日生成未来N天的动态排班，自动停诊处理

## 六、系统价值
本系统通过信息化手段优化就医流程，减少患者排队等待时间，提高医院号源管理效率，支持移动端与PC端多平台访问，具有良好的可扩展性（可对接支付、电子病历、短信通知等第三方服务）。
